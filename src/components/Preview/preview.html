Getting Started

Geno automatically generates a new directory called geno in the root of the project with all the files you need. Here is how you can create your first voice command:

  1. Add the following lines to an HTML file to include Geno:
    <link href="/PATH/TO/geno/geno.css" rel="stylesheet" />

    <script type="module" src="/PATH/TO/geno/custom.js"></script>
  2. Create a voice command.
  3. Build using the blue build button.
  4. Navigate to your application in a web browser.
  5. Start speaking to Geno by clicking the microphone button in the bottom right or Ctrl-`.


Creating Voice Commands

There are two types of voice commands you can create with Geno.

  1. Function commands trigger a function in a JavaScript file.
  2. Demo commands replay a sequence of clicks in the webpage.


Function Commands

You can add function commands to an existing function or use Geno to generate a function skeleton for you.

  1. Open a JavaScript file from the explorer pane on the left.
  2. In the editor, click the blue circle next to any function or click the + button in the top right to generate a function.
  3. Add a unique command name.
  4. Add sample user queries that trigger the command.
  5. Click each sample query and map words to any function parameters.
  6. Optionally configure multimodal input.
  7. Click "Create Function" if generating a new function, or ensure the pre-existing function is exported, e.g. export function foo()
  8. Click "Train Model" and wait for the model to finish training.
  9. Save your file and build using the blue build button.
  10. Test your voice command within your webpage.

Demo Commands

  1. Navigate to your webpage in the preview pane.
  2. Click the mouse cursor button in the menu bar.
  3. Record a sequence of clicks on clickable webpage elements. Typing into any text input element will generate a parameter that can be provided by the user to be automatically entered into the input element.
  4. Click the stop button in the menu bar of the preview pane.
  5. Finish configuring the command with queries and train the model.
  6. Build using the blue build button.


Configuring Multimodal Input

Multimodal input allows users to optionally provide a command parameter by hovering over a single element, dragging over multiple elements, or highlighting text.

  1. Open the popover for a command and find the "Mouse Context" section.
  2. Select the parameter to support multimodal input from the dropdown.
  3. Optionally specify a selector that should be recognized as context by clicking the crosshairs and hovering over an element in the webpage to select it.
  4. Optionally specify the element attribute(s) to use for the parameter's value. After the previous step, click the element in the popover and select the desired attribute(s).
  5. Configure the rest of the command options as needed, and build.


Adding Voice Responses

We have provided functions in Geno to add simple voice interaction functionality.

  • Import geno in your JavaScript file:

    import { geno } from "/PATH/TO/geno/geno.js";
  • geno.say(phrase, speak, callback) - speak to the user.

    phrase (string) - the phrase to say to the user
    speak (boolean) - generate speech from phrase and speak to user (defaults to true)
    callback (() => void) - callback to execute after saying phrase
